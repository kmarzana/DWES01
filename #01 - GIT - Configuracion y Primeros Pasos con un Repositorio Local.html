Git - Configuración y Primeros Pasos

Guía básica de configuración y uso inicial de Git

Entorno y Versión

Terminal: zsh | bash
% bash --> bash-3.2$
bash-3.2$ git --version
git version 2.39.5 (Apple Git-154)
NOTA

La versión de Git puede variar según el sistema operativo y la fecha de instalación.

Niveles de Configuración de Git

Git aplica las configuraciones en este orden (la última que lee prevalece):

1. Sistema (--system)

Afecta a todos los usuarios del sistema

2. Global (--global)

Afecta solo al usuario actual

3. Local (--local)

Afecta solo al repositorio actual

PRECEDENCIA

Las configuraciones locales anulan las globales, que a su vez anulan las del sistema.

Configuración Básica

Configuración del nombre de usuario:

bash-3.2$ git config --global user.name "kmarzana"

Configuración del mail de usuario:

bash-3.2$ git config --global user.email "kmarzana@birt.eus"

Configuración del editor por defecto:

bash-3.2$ git config --global core.editor "code --wait"
EDITOR

"code --wait" configura Visual Studio Code como editor. Asegúrate de tenerlo instalado y disponible en el PATH.

Configuración CRLF:

bash-3.2$ git config --global core.autocrlf true (Windows)
bash-3.2$ git config --global core.autocrlf input (Mac)
CRLF

Esta configuración ayuda a manejar correctamente los saltos de línea entre diferentes sistemas operativos.

Mostrar la configuración de Git:

bash-3.2$ git config --list
credential.helper=osxkeychain
init.defaultbranch=main
user.name=kmarzana
user.email=kmarzana@birt.eus
core.editor=code --wait
core.autocrlf=input

Primeros Pasos con un Repositorio Local

Inicializar repositorio:

bash-3.2$ git init
Initialized empty Git repository in /Users/kmarzana/Documents/GIT/DWES01/.git/

Estructura de carpetas:

bash-3.2$ ls -la
drwxr-xr-x 4 kmarzana staff 128 19 sep 00:14 .
drwxr-xr-x@ 5 kmarzana staff 160 18 sep 23:54 ..
drwxr-xr-x@ 9 kmarzana staff 288 19 sep 00:14 .git
-rw-r--r-- 1 kmarzana staff 1317 19 sep 00:09 GIT - Configuración y Primeros Pasos con un Repositorio Local.txt
CARPETA .git

La carpeta .git contiene toda la información de control de versiones. No modifiques manualmente su contenido.

Verificar estado del repositorio:

bash-3.2$ git status
On branch main

No commits yet

Untracked files:
(use "git add <file>..." to include in what will be committed)
"GIT - Configuracion y Primeros Pasos con un Repositorio Local.txt"

nothing added to commit but untracked files present (use "git add" to track)

Añadir archivos al Staging Area:

bash-3.2$ git add . (todos los archivos)
bash-3.2$ git status
On branch main

No commits yet

Changes to be committed:
(use "git rm --cached <file>..." to unstage)
new file: "GIT - Configuracion y Primeros Pasos con un Repositorio Local.txt"
STAGING AREA

El staging area (área de preparación) es donde se colocan los cambios que se incluirán en el próximo commit.

Realizar primer commit:

bash-3.2$ git commit -m "Commit inicial"
[main (root-commit) ff54b75] Commit inicial
1 file changed, 72 insertions(+)
create mode 100644 "GIT - Configuracion y Primeros Pasos con un Repositorio Local.txt"
MENSAJES DE COMMIT

Los mensajes de commit deben ser descriptivos y concisos. Es una buena práctica usar el imperativo ("Añadir" en lugar de "Añadido").
