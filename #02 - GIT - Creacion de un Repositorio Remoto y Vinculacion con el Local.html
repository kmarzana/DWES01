Conexión con Repositorio Remoto GitHub

Guía para vincular un repositorio local con un repositorio remoto en GitHub

Configuración del Repositorio Remoto

1

Crear un repositorio remoto en GitHub y copiar su URL HTTPS:

https://github.com/kmarzana/DWES01.git
NOTA

Asegúrate de que el repositorio remoto esté configurado como público o privado según tus necesidades.

2

Vincular el repositorio local con el repositorio remoto:

bash-3.2$ git remote add origin https://github.com/kmarzana/DWES01.git
ALIAS "ORIGIN"

origin es el alias estándar que se utiliza para referirse al repositorio remoto principal al que está conectado el repositorio local.

3

Subir los commits del repositorio local a la rama main remota:

bash-3.2$ git push -u origin main
Enumerating objects: 4, done.
Counting objects: 100% (4/4), done.
Delta compression using up to 8 threads
Compressing objects: 100% (4/4), done.
Writing objects: 100% (4/4), 3.88 KiB | 3.88 MiB/s, done.
Total 4 (delta 0), reused 0 (delta 0), pack-reused 0
To https://github.com/kmarzana/DWES01.git
* [new branch] main -> main
branch 'main' set up to track 'origin/main'.
PARÁMETRO -U

El parámetro -u (o --set-upstream) establece una relación de seguimiento entre la rama local y la rama remota, permitiendo usar git push sin especificar la rama remota en el futuro.

4

Subir commits posteriores:

bash-3.2$ git push
SIMPLIFICACIÓN

Después de configurar la relación de seguimiento con -u, Git recordará la rama remota asociada, permitiendo usar simplemente git push para subir cambios futuros.

Comandos Adicionales Útiles

Verificar la conexión remota configurada:

bash-3.2$ git remote -v
origin https://github.com/kmarzana/DWES01.git (fetch)
origin https://github.com/kmarzana/DWES01.git (push)

Cambiar la URL del repositorio remoto:

bash-3.2$ git remote set-url origin https://github.com/kmarzana/NUEVO-REPO.git

Eliminar la conexión con un repositorio remoto:

bash-3.2$ git remote remove origin
